Overview

My first project was working with a Lan Turtle from Hak5. These are great for man in the middle reconnaissance and attacks. When purchasing one it comes with instructions on how to get it setup, and there is plenty of documentation in other places as well. What I will be talking about is the capabilities it has when coupled with a SSH tunnel. To start off, I used Linode because they offer a 100$ credit when signing up for their service: you can get 10 months free from the start with Linode. The tool that is important when trying to SSH with windows is PuTTY, and its free. First you must SSH into the Linode server so that you can create a user for the Lan Turtle to connect to. Once that is done you can then SSH into the Lan Turtle and create an Authentication token. What this token will be used for is so that on boot the token will be used to authenticate it with the Linode server however you have to manually SSH from the Lan Turtle to the Linode server so that the interaction can be confirmed, its like a challenge-response between the two. The hard work is over, now you can SSH into the Linode server and make sure to sign into the new user, and from there SSH into localhost and port 22 because the Lan turtle is connected to the new user from a different port.

Another project I started on was using the Bash Bunny tool It is similar to the Rubber Ducky, which is more widely known. Just like the name suggests the Bash bunny is a powerful tool. I chose not to do anything too harmful and more of a practical gag than a serious hack. To keep it brief I wrote a script in which the bash bunny creates a .bat file (batch file) and executes it during every startup. I accomplished this by saving it in the startup directory of Windows. What this script does is executes a PowerShell command stored elsewhere. There are two reasons I did this. First because executing a PowerShell on startup brings out an unwanted script confirmation dialogue. Secondly, the batch file contains text which is displayed using the echo command on cmd. The code is filled with backslashes because of the problem with Linux Interpreting them differently that Windows does.
