Overview

The Google IT certificate is obtained through a 5 course certificate program and has hundreds of examples of virtual assessments that include: computer assembly, networking simulation, writing effective documentation, routing paths and subnets, wireless networking, device software management, public key or asymmetric encryption, incident handling, and more.

The program is ACE© certified which mean 12/4 associate degree-level courses have equal value to it.

It took me about 4 months to complete it. The proposed time frame is 6 months with about 5 hours of engagement a day.

In the first course I learned about the fundamenetals of computers. This is more of a brief introduction of what I am going to learn. I am introduced to

  • How binay works
  • How to assemble a computer
  • Operating systems
  • Understading the internet and how it works
  • Common soft skills

The second course went a bit more in depth with how newtorks operate.

  • The 5-layer model of networks
  • Protocols associated with TCP/IP
  • Troubleshooting techniques
  • Services like DNS and DHCP
  • Understading cloud computing, EaaS, and cloud storage

Hopping into the third course I am greeted by Operating Systems

  • Navigating through Linux and Windows
  • Creating users, groups, files, and permission access
  • Configuring partitions and filesystems
  • Understanding how system processes work
  • Work with system logs and remote connection tools
  • Knowlegde for common issues others encounter

The 4th course is all about Infrastrcure

  • Best practices for hardware, vendors, and services for your organization
  • Understanding how Infrastrcure services keep the organization running
  • Understand the importance of the cloud
  • How to manage computers in a network
  • Managing computer using directory services, Active Directory, and OpenLDAP
  • Backup for disaster recovery and plans to improve processes for IT

The final course is an in-depth look at security

  • How encryption algorithms work
  • Various authentication systems and the types
  • Difference between authentication and authorization
  • Evaluate potential risks and ways to reduce
  • Best practices and concepts to help others protect themselves and learn
